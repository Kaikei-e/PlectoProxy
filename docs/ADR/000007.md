# OCI artifact + wkg で配布し、宣言的マニフェストの content hash で無停止リロードを整合させる

## Status
accepted

## Date
2026-06-20

## Affected Components
- distribution — フィルタを OCI artifact として配布、`wkg` で fetch/publish、digest 固定
- host runtime / control — 宣言的マニフェスト、無停止リロード手順

## Context
フィルタと設定の配布・バージョニング・ホットリロードの方式を決める。CNCF TAG Runtime が Wasm OCI Artifact レイアウト
（config mediaType `application/vnd.wasm.config.v0+json`、layer mediaType `application/wasm`）を定義し、Bytecode Alliance の
`wkg`（wasm-pkg-tools）が fetch/publish を担い、`wkg.lock` で content digest を固定でき、cosign 署名や SBOM など既存の
OCI 投資を流用できる。Envoy は「拡張を control plane から runtime にリロードできる」agility を proxy-wasm の主要価値に挙げる。
Plecto は重い control plane を持たない（[[000008]]）。

## Decision
- 配布は **OCI artifact + `wkg`**。フィルタ参照は OCI ダイジェストで固定し、`wkg.lock` 相当のロックファイルで再現性を担保。
- フィルタと設定のバージョンは **単一の宣言的マニフェスト**（ルート・チェーン定義・フィルタ OCI 参照＋ダイジェスト）で
  **content hash により整合をロック**する。
- 無停止差し替えは **「新インスタンスを並行生成 → アトミックに切替 → 旧インスタンスを drain」**。

検討した代替案と却下理由:
- **独自配布フォーマット**: OCI の署名/SBOM/レジストリ等の既存投資を捨てる。却下。
- **ダイジェスト非固定（タグ運用）**: 再現性が壊れ、サプライチェーン整合を担保できない。却下。

本決定はファウンディング設計の Fork 8 に対応する。

## Consequences

### Pros
- 既存の OCI エコシステム（レジストリ・cosign・SBOM）をそのまま流用できる。
- content hash 固定で「何がロードされているか」が再現可能・監査可能。
- 公式・未改変のホストでカスタム拡張をロードでき、無停止で差し替えられる。

### Cons / Tradeoffs
- マニフェストと OCI ダイジェストの整合運用（更新時のロック更新）が要る。
- **再検討条件**: フィルタ数が数百規模になり静的マニフェスト管理が破綻したら、動的配布レイヤを段階導入する（[[000008]]）。

## 実装ノート（確定事項・2026-06-20）

本決定を実装した際に固まった具体判断。新クレート `plecto-control`（[[000008]] の control plane の具体化）として
起こし、**done** と **named-deferred** を区別する。ADR7 は配布・マニフェスト・無停止リロードの三本を立てたが、
fast-path server が未着手のため、実装可能な範囲と繰延を仕分けた。

### done

1. **新クレート `plecto-control`。** `manifest`（TOML）/ `artifact`（store seam）/ `oci`（offline layout）/
   `chain`（dispatcher）モジュール。`plecto-host` に依存する上位層で、single-node-first（[[000008]]）。

2. **マニフェストは単一の TOML 宣言（[[000008]] 静的設定）。** `[trust] keys`（信頼鍵 PEM パス、[[000006]]）/
   `[[filter]]`（id・source・digest pin・isolation・任意 deadline/memory）/ `[chain] filters`（順序付き id 列）。
   `deny_unknown_fields` で typo を fail-closed。ルート・named chain は fast-path server まで繰延し、v0.1 は単一 chain。

3. **OCI ソースは local OCI image-layout（オフライン）で、remote registry pull ではない。** ADR7 本文の wkg fetch
   （network）と [[000008]] single-node/self-host・[[000006]] offline-keyed の offline 姿勢が緊張する。これを
   **host は local layout を読み、remote fetch（wkg）は operator の out-of-band ステップ**とする形で解いた。CNCF
   Wasm OCI Artifact layout（config `application/vnd.wasm.config.v0+json`、layer `application/wasm`）を読む。

4. **実装は `oci-spec` + `sha2` + `std::fs` の hand-rolled（`ocidir` 不採用）。** `ocidir` は batteries-included だが
   `openssl`（C）+ `tokio`（dev）+ `flate2`/`tar` を引き込み、Plecto の no-openssl / no-tokio / minimal-deps 制約と
   衝突する。`oci-spec`（pure-Rust・no tokio/openssl）が spec-correct な型を与え、layout の write/read と sha256
   検証は自前で持つ（fail-closed ゆえ digest 検証は元々自分で要る）。**lock-in**: OCI 型は `oci-spec` に固定。

5. **cosign 署名と SBOM は同一 artifact 内の bundled custom-mediaType layer。** `application/vnd.plecto.signature` /
   `…sbom` / `…sbom.signature` を wasm layer と並べる。[[000006]] の sign-blob（detached 署名）に自然で、単一 artifact が
   component+sig+sbom を自己完結で運ぶ。cosign-standard の referrers / `.sig`（別オブジェクト参照）は offline layout
   と相性が悪く不採用。

6. **digest 固定 = content-hash ロック（wkg.lock 相当）。** マニフェストは各 filter を OCI image-manifest の sha256 で
   pin し、ロード時に (a) pin 照合・(b) 各 blob の content digest 照合の二段で検証（fail-closed）。これは [[000006]] の
   署名（authenticity）とは別レイヤの integrity / reproducibility で、両者が独立に効くことをテストで固定した
   （tampered component は digest pin が自己整合でも署名で落ちる）。

7. **`ArtifactStore` trait seam（host の `KvBackend` と同形）。** `OciLayoutStore`（offline）と `MemoryStore`
   （reference / test）。将来の remote / wkg fetcher は manifest / chain / reload に触れず差し込める。

8. **chain dispatcher は完全 semantics。** request 側を順に駆動し `continue` / `modified`（**edit を適用**）/
   `short-circuit`、response 側は逆順（CONTEXT の対称性）、chain 内の `RunError`（trap / deadline）は **fail-closed**
   （5xx・upstream 非到達）。host は従来 decision を返すだけで edit を適用していなかった——dispatcher が初めて適用する。
   出力は `ChainOutcome { Forward(request), Respond(response) }`。

9. **無停止リロードは明示 API `Control::reload(manifest)` + ArcSwap。** 新 set を完全ビルド→アトミック swap→旧は
   Arc drop で drain。all-or-nothing（失敗時は旧 set 据え置き）。trust policy は構築時固定で、reload は同一 Host
   （同一 epoch ticker、[[000006]]）上で filter set + chain だけ差し替える——runaway フィルタは reload を跨いでも bounded。

10. **`Control::from_manifest` を ops エントリポイントに。** `[trust] keys` PEM を読み Host を構築、`base_dir` 下の
    OCI layout から filter を解決してロードする。マニフェスト一枚で trust root・filter・chain が完結する（ADR7 の
    「単一宣言的マニフェスト」）。

11. **WIT 契約は不変。** manifest / OCI / chain / reload はすべて host の外（control 層）で、`plecto:filter` ワールドを
    一切変えない。

### named-deferred

- **remote registry pull / wkg fetch**（operator の out-of-band ステップ。`OciLayoutStore` は wkg が生成した layout を読む）。
- **ルート + named chain + route 照合**（fast-path server 到来時）。
- **drain（in-flight リクエスト追跡）**（fast-path server が並行負荷を載せてから。現状は ArcSwap の Arc drop が論理 drain）。
- **file-watch / SIGHUP の reload トリガ**（v0.1 は明示 API）。
- **content-hash ロックファイルの外部化**（v0.1 は pin 済み digest がロックそのもの。別ファイル化は配布運用が要れば）。
