# 多言語例フィルタを zero-WASI 制約下で提供し、fat guest への最小 WASI 貸与を別判断に繰り延べる

## Status
accepted

## Date
2026-07-06

## Affected Components
- examples/filters — `filter-hello-moonbit`（約 22 KB）/ `filter-hello-js`（ComponentizeJS、約 12 MB）/ `filter-hello-c`（wasi-sdk、約 66 KB）。いずれも filter-hello の契約行使サブセットの移植で、`build.sh` が `wasm-tools component wit` の出力に `wasi:` が現れたらビルドを落とす
- host runtime — `polyglot-conformance` feature と `crates/host/tests/polyglot.rs`。既定の `cargo test` には影響しない。Linker・provenance ゲートは一切変更なし
- CI — `polyglot-guests` job。wasm-tools 1.252.0 / wit-bindgen 0.58.0 / wasi-sdk-33 は sha256 pin、MoonBit は versioned artifact 未配布のため latest（要フォロー）

## Context

README と CLAUDE.md は「任意言語でフィルタを書ける」（polyglot）を Plecto の強みとして掲げてきたが、
実際に動く guest は Rust のみで、README の Go / JS / Python への言及は未検証だった（M6 も
"not started"）。2026-07 の調査（一次ソース: component-model.bytecodealliance.org、各ツールチェーンの
README / リリース）と実ビルドで、言語ごとの現実は次のとおり確定した:

- **zero-WASI コンポーネントを今日出せる**: MoonBit（wit-bindgen in-tree、adapter 不要、実測 21.5 KB）、
  C（wasi-sdk `wasm32-wasip2-clang -mexec-model=reactor`、WASI API を呼ばなければ import ゼロ、実測
  67.6 KB）、JavaScript（ComponentizeJS 0.21 の `disableFeatures` で "pure component"、実測 12.6 MB）、
  Python（`componentize-py --stub-wasi`、約 17 MB）。
- **出せない**: Go。TinyGo の `wasip2` ターゲットは `wasi:cli/command` world 前提で、成果物は必ず
  `wasi:cli` / `wasi:io` / `wasi:clocks` を import する。big Go（gc）は Component Model 未対応
  （wasip3 port は proposal 段階、projected）。
- ポリグロットを可能にしているのは **Component Model + WIT 契約**であり WASI ではない。むしろ現行
  フィルタの性質（小サイズ・deny-by-default Linker が WASI を一切リンクしない）は [[000010]] の
  「WASI を持ち込まない」判断の帰結。WASI 0.3.0（2026-06-11 リリース、確定事実）を guest 側で書ける
  ツールチェーンは実質 Rust nightly（`wasm32-wasip3` Tier 3）と JS のみで、本判断には影響しない。

つまり「多言語例を追加する」タスクは、**host の Linker に最小 WASI を足すか否か**という
セキュリティ判断（[[000010]] が「wasip2 移行時の新セキュリティ表面」として名指しで繰り延べた課題）と
不可分だった。

## Decision

**host を一切変えず、zero-WASI コンポーネントを出せる言語（MoonBit / JavaScript / C）だけを
例として提供する。** 3 例は filter-hello の契約行使サブセット（init カウンタ / host-log /
block 403 / typed edit / host-native ratelimit 429 / body uppercase・deny / respedit）の忠実な移植とし、
`crates/host/tests/polyglot.rs` が **同一のアサーション群**を 3 コンポーネントに回す。CI の
`polyglot-guests` job がビルド → WASI import ゼロ検査 → conformance を毎 PR で実行し、polyglot 主張を
反証可能な形に保つ。ドキュメント（README / writing-a-filter §7 / ROADMAP / CONTEXT.md 用語
「Zero-WASI guest / Fat guest」）を検証済みの事実へ更新した。

検討した代替案と却下理由:

1. **host に feature-gate で最小 WasiCtx リンカを追加し、TinyGo（fat guest）も例に含める** —
   Go デモが得られる代わりに、[[000010]] が繰り延べた新セキュリティ表面（std を満たす最小 `WasiCtx` の
   scope 設計、over-grant リスク）を「例を増やす」という動機で開くことになる。deny-by-default
   （コア原則）に関わる判断を例の副作用として行うべきではなく、別 ADR に切り出して繰り延べ。
2. **Python 例も追加** — `--stub-wasi` で同じ制約下に収まるが約 17 MB とフィルタ用途に過大で、
   stub された WASI を stdlib が踏むと実行時 trap するリスクもあり、デモ価値がコストに見合わない。
   writing-a-filter §7 の記述のみに留めた。
3. **ビルド済み .wasm を commit して CI をツールチェーン無しにする** — JS の約 12 MB バイナリが
   リポジトリに入り、ソースと成果物の乖離・サプライチェーン監査性（[[000006]] の姿勢と不整合）の
   問題があるため却下。CI は sha256 pin したツールチェーンで毎回ソースからビルドする。
4. **調査レポートのみで例は追加しない** — README の未検証な polyglot 主張が残り続ける。主張は
   実行される conformance で担保するのが本プロジェクトの流儀（consumer-driven contract test）。

本判断は Tenet「deny-by-default capability」をターゲット非依存の形で具体化する（[[000010]] の
「セキュリティ境界の実体は Linker でありターゲット非依存」という根拠の、多言語での実証）。また
[[000002]] / [[000020]] の polyglot 方向（M6）の最初の実装着地となる。

なお MoonBit は文字列が UTF-16 のため `wasm-tools component embed --encoding utf16` が必須
（欠くと文字列がすべて壊れるが instantiate 自体は成功する — conformance が無ければ静かに壊れる好例）。

## Consequences

### Pros
- polyglot 主張が「aspirational」から「CI が毎 PR 反証を試みる事実」になった。契約（WIT）の
  言語中立性・Linker のターゲット非依存性・provenance ゲートの言語非依存性が同一テストで担保される
- host のデフォルト経路（Linker / 供給鎖 / ビルド）は 1 バイトも変わらない。既定の `cargo test` も
  非 Rust ツールチェーンを要求しない（feature gate）
- 例の非対称性（Go が無い理由）がドキュメントで明示され、将来の fat guest 判断への入力が揃った

### Cons / Tradeoffs
- CI job が 3 ツールチェーン（+ Node）に依存し、MoonBit だけ latest 追随（versioned artifact 未配布）。
  MoonBit の言語・wit-bindgen backend は動きが速く、CI が上流更新で割れうる
- MoonBit の生成バインディングは commit 運用（`gen/world/filterBody/moon.pkg.json` に手動 import 追加が
  要るため）。WIT 変更時は再生成 + 手直しの二段作業になる（build.sh 冒頭に手順を明記）
- JS 例は 12.6 MB。conformance の untrusted ロードが遅く、polyglot テストは約 100 秒かかる
- **再検討条件**: (1) fat guest（TinyGo/Go）需要が実際に立ったとき — 最小 WasiCtx リンカを別 ADR で
  判断（[[000010]] の繰延べ課題の解消）。**（2026-07-06 更新）**(1) は [[000063]] で判断済み——Tier B
  （最小 WASI 貸与、stdout/stderr は host-log ブリッジ）として Go を解禁する。(2) M3 Stage 2 の wasi:http 収斂ゲート（[[000020]]）が
  通ったとき — 例の契約・型語彙の追随。(3) MoonBit が versioned artifact を配布したとき — CI の pin。
  (4) TinyGo が zero-WASI reactor 出力を得る／big Go の wasip3 port が着地したとき — Go 例の再評価
