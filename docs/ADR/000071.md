# HTTP ヘッダ値を list<u8> に持ち上げ、plecto:filter@0.2.0 を契約-first で導入する

## Status
accepted（2026-07-10 実装・proxy E2E / host E2E green）

## Date
2026-07-10

## Affected Components
- wit — `plecto/wit/v0.1.0/world.wit`（凍結）、`plecto/wit/v0.2.0/world.wit`（`header.value: list<u8>`）、開発デフォルトの `plecto/wit/world.wit` は 0.2.0
- host — `contract.rs`（バージョン検出・0.1 アダプタ・guest 出力ヘッダ検証）、`runtime.rs`（デュアル `FilterPre`）
- server — `headers.rs`（ingress/egress とも `Vec<u8>` 原文、`HeaderValue::from_bytes`）
- examples/filters — スキャフォールド・in-tree フィルタを 0.2 へ移行。polyglot 0.1 コンポーネントはロード時アダプタで継続（deprecation warning 1 回）

## Context

敵対的レビュー（2026-07-10）で、fast path が hyper → 契約へ投影する段階で `from_utf8_lossy` により **非 UTF-8 ヘッダ原文がフィルタに届かない** ことが確定した。filterless 経路の `copy_headers_preserving` はホスト側の暫定措置であり、WIT が `string` のままではフィルタが原文を観測・判断できない（P3 の型で運ぶ、という原則に反する）。

`wasi:http` 型への即時収束（[[000020]]）は wasm32-wasip3 / streaming 証拠トラックが未達のため今回の範囲外。record / decision の形は維持し、**ヘッダ値だけ** `list<u8>` へ上げる minor（pre-1.0 では breaking になり得る）を採用する。

## Decision

1. **`plecto:filter@0.2.0` を契約正文として追加する。** `header.value` のみ `list<u8>`。`filter` / `filter-body` ワールド、host-API（`host-config` 含む）は 0.1 と同形。0.1 は `plecto/wit/v0.1.0/` に凍結し、ロード可能（アダプタ付き）。
2. **ホストの canonical `Header` は `Vec<u8>`。** 0.2 guest は原文を受け取る。0.1 guest には呼出時のみ lossy 投影し、`continue` では native 原文を再構成しない（明示 `modified` の set は UTF-8 バイトとして適用）。
3. **fast path の lossy 一致ヒューリスティックを撤去する。** ingress は `HeaderMap` → `Vec<u8>` 原文、egress は `HeaderValue::from_bytes`。guest が返したヘッダは CRLF・不正 name/value・上限を検証し、違反は trap ではなく fail-closed（`RunError`）。
4. **互換と移行:** 新規・in-tree フィルタのデフォルトは 0.2。0.1 ロード時は stderr に 1 回 deprecation warning。semver 1.0 前は minor が breaking になり得ることを本文に明示する。

検討した代替案と却下理由:

- **ホストのみ byte 保持・WIT は 0.1 のまま** — フィルタが原文を契約上観測できず、polyglot / conformance の falsifiable claim が成立しない。却下。
- **即 `wasi:http` fields へ収束** — P3 toolchain / streaming ゲート未達（[[000020]] / [[000068]] 序列）。却下（別スライス）。
- **semver 1.0 へ一気に跳ばす** — プロジェクト成熟度が不足（ユーザー合意）。0.2 系で段階導入。

## Consequences

### Pros
- 非 UTF-8・同一 lossy 表現に潰れる異なる byte 列を、フィルタ境界を含めて end-to-end で扱える（proxy E2E + host E2E で固定）。
- 0.1 資産（polyglot fixtures）を無停止で載せ替え可能。

### Cons / Tradeoffs
- pre-1.0 のため全フィルタの再ビルドが必要（0.1 は警告付き猶予）。
- デュアル bindgen / アダプタがホストに常駐（[[000071]] 実装後、0.1 退役で削減可能）。

**再検討条件:** (a) `wasi:http` 収束スライスで header 型を再統合する、(b) 0.1 ロードが 6 ヶ月ゼロになったらアダプタ削除を ADR 化する。
