# Roadmap

Plecto Proxy is built ADR-first; each milestone realizes specific design decisions in
[`docs/ADR/`](ADR/). This is the detailed, milestone-by-milestone view — the
[README](../README.md#roadmap) carries only a compact status table with a link back here.

- **M0 — Foundation** ✅ *(done)*
  The current contract is **`plecto:filter@0.3.0`** (byte-valued headers, [ADR 71](ADR/000071.md); response-side request context + `replace` decision, [ADR 73](ADR/000073.md)); `0.1.0` / `0.2.0` remain frozen at `wit/v0.1.0/` / `wit/v0.2.0/` with load-time adapters. A wasmtime host loads & runs filters behind a deny-by-default capability boundary (log / clock / kv / counter / rate-limit / config), with an example filter, E2E/conformance/unit tests, and CI. — [ADR 1](ADR/000001.md) · [2](ADR/000002.md) · [10](ADR/000010.md) · [66](ADR/000066.md) · [71](ADR/000071.md)

- **M1 — Filter runtime hardening** ✅ *(landed)*
  Trust-branched instances: trusted filters reuse a fixed-capacity, lazily-filled **pool** checked out per request (bounded-wait fail-closed, pool-wide circuit breaker, recycle-after-N); untrusted run **fresh-per-request** (linear memory fresh by construction). redb-backed host KV + atomic counters + **host-native token-bucket rate limiting** (bucket spec host-set in the manifest, keyed per request — an untrusted filter cannot widen its own limit, [ADR 26](ADR/000026.md)), per-filter key namespacing, **per-filter + host-wide quotas** on that state (fail-closed over quota, untrusted `init` deadline tightened — [ADR 27](ADR/000027.md)), and **epoch metering + memory/table limits**. The trusted/untrusted split is *forced* (not just perf) by the init/zeroization knot.

- **M2 — The data path (fast path)** 🚧 *(matured by the role-driven model of [ADR 29](ADR/000029.md))*
  A tokio + hyper + quinn `plecto-server` that terminates **HTTP/1.1, HTTP/2 (ALPN) and HTTP/3 (QUIC)** and **TLS** (rustls, manifest-declared certs, fail-closed), **routes** by host · path-prefix · method · header · query in specificity order with weighted **traffic split / canary** ([ADR 34](ADR/000034.md)), runs the route's filter chain via a `spawn_blocking` bridge to the M1 pool, and **load-balances** across healthy upstream instances with a per-upstream choice of **round-robin**, **weighted least-request (P2C)**, or **weighted Maglev** consistent hashing ([ADR 35](ADR/000035.md)). Resilience is layered on the same upstream: active/passive **health checks** (pessimistic start, fail-closed 503 when none healthy), **outlier detection** ([ADR 32](ADR/000032.md)), a per-upstream **circuit breaker** ([ADR 28](ADR/000028.md)), **two-tier timeouts** (per-try + overall, [ADR 31](ADR/000031.md)), **bounded retry** with jittered backoff and retry-on-5xx ([ADR 30](ADR/000030.md)), and a native L7 **rate-limit** floor per route / client-IP ([ADR 33](ADR/000033.md)). Edge-model client-IP propagation re-issues `X-Forwarded-For` / `X-Real-IP` from the real peer ([ADR 18](ADR/000018.md) / [22](ADR/000022.md)). The data path is **security-hardened** ([ADR 27](ADR/000027.md)): the request path is normalized once at ingress so a per-route filter is a reliable auth boundary (encoded separators / dot-escapes rejected fail-closed), inbound resource limits are enforced (connection cap, header-read timeout, body-read deadline, body-buffer budget), and filter-untouched header bytes pass through **byte-for-byte**. Upstream legs can be **re-encrypted with TLS+ALPN** — HTTP/2 preferred, HTTP/1.1 fallback, optional custom CA, `TE: trailers` passed through and forwarded so gRPC runs end to end ([ADR 42](ADR/000042.md)); hostname upstreams can be **periodically re-resolved**, turning every A/AAAA record into its own load-balancing endpoint that tracks container re-creation ([ADR 44](ADR/000044.md)). A per-route **HTTP/1.1 `Upgrade` token allowlist** (`h2c` rejected at validation) re-issues the handshake upstream and splices a bidirectional tunnel with an activity-based idle timeout, so WebSocket now runs end to end ([ADR 48](ADR/000048.md)). The data-plane bind address and Alt-Svc advertised port are declared in a `[listen]` manifest section, fixing container `0.0.0.0` binds and internal-vs-published port mismatches ([ADR 45](ADR/000045.md)). Opt-in **PROXY protocol v2** reception (`[listen.proxy_protocol]`, trusted-CIDR-required, fail-closed, the pure parser is the first cargo-fuzz target) restores the real client identity behind a fronting L4 LB, so the per-client-IP rate limit, `SourceIp` Maglev, `X-Forwarded-*` re-issuing and the access log keep their semantics across N replicas ([ADR 57](ADR/000057.md)). *Next:* mTLS (downstream + upstream client certs), HTTP/3 graceful drain (GOAWAY), then response compression; EWMA/latency-aware LB, ring-hash hashing, header-presence / regex match remain further out — see [ADR 43](ADR/000043.md) for the declined items (response caching, a native AI/LLM gateway) and [ADR 56](ADR/000056.md) for the current deferred order and the two-program (multi-replica readiness / filter ecosystem) roadmap framing. All host-held state — the rate limiter above, `host-kv`/`host-counter`/`host-ratelimit`, and the redb backend — is **node-local**; native cross-replica distributed state is declined in favor of documented front-LB + N-replica operational patterns ([ADR 53](ADR/000053.md), [hardening guide](hardening.md)).

- **M4 — Provenance & zero-downtime reload** ✅ *(landed)*
  OCI-artifact filter distribution (offline image-layout, digest-pinned) + cosign signature verification + SBOM↔component binding, and content-hash-reconciled hot reload from a declarative manifest (atomic `ArcSwap`, all-or-nothing, SIGHUP-driven). The shipped `plecto` binary now wires that reload loop directly — SIGHUP triggers it, SIGTERM/SIGINT drain in-flight requests before exit ([ADR 39](ADR/000039.md)) — and `plecto validate` runs the same fail-closed manifest checks without loading any artifact, for CI and pre-reload gating ([ADR 46](ADR/000046.md)). The same supply-chain discipline now applies to Plecto Proxy's own binaries and container image: `cargo-auditable` builds, an SBOM (syft), and a cosign keyless signature ship with every tagged release — `v0.1.0` is the first tag ([ADR 47](ADR/000047.md)). What remains is a *remote* filter-registry fetch path (the `wkg` boundary, out-of-band by design). — [ADR 6](ADR/000006.md) · [8](ADR/000008.md)

- **M5 — Observability & opt-in distribution** 🚧 *(span/metrics/export landed; opt-in distribution deferred)*
  **Landed:** host-propagated W3C trace context (inbound `traceparent` continued through the proxy), one span per filter execution over the OpenTelemetry data model, a sync `TelemetrySink` (in-memory + host-aggregated RED metrics), and **OTLP network export** — a host-side export pump (batch/retry/flush) ships spans over the wire without touching the guest contract, so the no-tokio filter boundary stays intact ([ADR 40](ADR/000040.md)). **Deferred:** opt-in `foca`/`openraft` config consensus. — [ADR 7](ADR/000007.md) · [9](ADR/000009.md) · [40](ADR/000040.md)

- **M3 — Async & bodies** 🚧 *(Stage 1 landed; Stage 2 landed; experimental streaming increment gated on wasip3)*
  The frontier, since M4/M5 are largely done. **Stage 1 (landed):** [wasmtime 46](https://github.com/bytecodealliance/wasmtime/releases/tag/v46.0.0) (2026-06-22) made WASI 0.3 + Component Model async default-on; the host runs guest hooks via `call_async` on wasmtime fibers, bridged to its still-sync public API with `block_on`. **Stage 2 (landed):** the request-side **body hook** is wired **end-to-end** — `on-request-body` (buffer-then-decide; body as a buffered `list<u8>` in v1) through the contract, the host, **and the fast path** (the proxy buffers a filtered route's body bounded — 16 MiB cap, fail-closed 413), conformance- and E2E-green ([ADR 25](ADR/000025.md)). The contract is split into two worlds — header-only `filter` and body-reading `filter-body` — so the **absence** of `on-request-body` is itself what tells the host to skip buffering: bodyless requests, filter-less routes, **and now routes whose filters are header-only** all stay zero-copy; only a route running an actual `filter-body` filter pays the buffer cost ([ADR 38](ADR/000038.md)). An **experimental, feature-gated** `stream<u8>` increment has landed: behind the off-by-default `streaming-body` feature, a separate `plecto:filter-streaming` world drives an async `process-body(stream<u8>)` on the host (no whole-body buffer; server-side wiring is the next increment) over a minimal WASI slice, and stays out of the default build until `wasm32-wasip3` reaches Tier-2. Header **byte-equivalence** through the proxy (filter-untouched bytes preserved) landed alongside. [ADR 20](ADR/000020.md) keeps the `wasi:http` convergence direction — deny-by-default independent of the type vocabulary.

- **M6 — Polyglot SDKs & reference filters** 🚧 *(polyglot examples + conformance CI landed; the outbound capability has landed feature-gated)*
  **Landed (2026-07):** the filter-hello conformance subset ported to **MoonBit / JavaScript / C** (`plecto/examples/filters/filter-hello-{moonbit,js,c}`), each a **zero-WASI** component the unchanged deny-by-default Linker instantiates, all driven through ONE shared assertion suite (`crates/host/tests/polyglot.rs`, feature `polyglot-conformance`) by the `polyglot-guests` CI job — the polyglot claim is now falsifiable, not aspirational. Python is possible the same way (`componentize-py --stub-wasi`) but too heavy to bundle. **Go/TinyGo** is the first **Tier B fat guest** behind the off-by-default `fat-guest` feature and manifest `wasi = "minimal"` ([ADR 63](ADR/000063.md)): a fixed minimal WASI slice (`wasi:io` / `clocks` / `random` / `cli`, still zero filesystem and zero sockets), verified by `polyglot_tier_b.rs` / CI job `polyglot-guest-go` (`filter-hello-go`).
  Still pending: Go / JS / Python filter **templates** (beyond one fixture per language) and broader reference auth / rate-limit / WAF filters. The capability surface a filter can be lent is expanding deny-by-default: **outbound HTTP has landed behind the off-by-default `outbound-http` feature** ([ADR 36](ADR/000036.md)) — a filter can make ext_authz / JWKS / token-introspection / OPA calls through `wasi:http/outgoing-handler`, gated by an operator per-filter allowlist and an **IP-pinned SSRF guard** (every resolved address classified, DNS-rebinding closed), fail-closed and resource-bounded; it stays out of the default build until the `wasi:http` convergence gate. WAF is deliberately placed in the extension plane rather than native ([ADR 37](ADR/000037.md)).
